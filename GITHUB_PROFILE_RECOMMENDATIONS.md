## Public README baseline

For public-facing repos, use this compact structure:

```markdown
# Project Name

A short description of what it is.

## Why I built it

Explain the problem it explores or solves.

## What it does

- Use 3 to 6 concrete bullets.

## Status

Active prototype, Work in progress, Archived experiment, or Reference project.

## Notes

Mention the broader AI-assisted design workflow, design systems, or web tooling context when accurate.
```
